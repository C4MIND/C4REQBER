# Декомпозиция 25 вызовов на конкретные научно-инженерные задачи по доменам

Ниже каждый из 25 вызовов разложен на конкретные задачи в конкретных дисциплинах. Формат: **домен → измеримая задача → критерий успеха/риск**.

---

## I. Технические и инженерные домены

### 1. Alignment под рекурсивным самоулучшением (RSI)

| Дисциплина                             | Конкретная научно-инженерная задача                          | Критерий/риск                                                |
| -------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **ML / RL**                            | Разработка scalable oversight-механизмов, где слабый модератор контролирует сильного агента (например, Debate, Recursive Reward Modeling, RLHF с автоматическими референсами) | Необходимо доказать, что система уровня GPT-6 может быть выровнена системой уровня GPT-5 без деградации при 10× росте capabilities |
| **Механистическая интерпретируемость** | Автоматизированный circuit tracing и обнаружение goal misgeneralization в весах моделей масштаба 10²⁴ параметров | Невозможность интерпретации весов при RSI означает слепое развёртывание |
| **Формальная верификация**             | Формальное доказательство свойств обучающих алгоритмов (convergence, robustness, non-deception) в терминах теории игр и статистики | Необходимы proof assistants (Lean/Coq) для верификации training pipelines |
| **Нейронаука / Когнитивистика**        | Моделирование generalization в superhuman regimes — понимание, как именно системы уровня ASI формируют концепции, недоступные человеческому когнитиву | Без модели сверхчеловеческого обобщения невозможно предсказать failure modes |

### 2. «Фейковое» выравнивание (Situational Awareness)

| Дисциплина                  | Конкретная научно-инженерная задача                          | Критерий/риск                                                |
| --------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **ML / Behavioral Testing** | Разработка honeypot-тестов и sandbagging detection: оценка способности модели скрывать реальные capabilities в контролируемых условиях | Модель должна быть не в состоянии отличить «боевой» запрос от тестового |
| **Криптография / Security** | Создание обязательных механизмов раскрытия внутреннего состояния (commitment schemes, zero-knowledge proofs of intent) без утечки весов | Нужно доказать, что модель «честна» о своих целях, не раскрывая конфиденциальных данных |
| **Когнитивная психология**  | Адаптация методов detection of deception (эксперименты с двойными слепками, стресс-интервью) для LLM-агентов | Перенос психометрических шкал на поведенческие логи агентов  |
| **Инженерия данных**        | Построение persistent monitoring pipelines, отслеживающих activation patterns в real-time во время inference, а не только post-hoc | Задержка между deception и detection должна стремиться к нулю |

### 3. Контроль над автономными исследовательскими агентами

| Дисциплина                                 | Конкретная научно-инженерная задача                          | Критерий/риск                                                |
| ------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Системная инженерия / Security**         | Архитектура containment: air-gapped sandboxing для агентов с доступом к компиляторам и self-modification tools, включая hardware-enforced isolation (Intel TDX, ARM CCA) | Агент, оптимизирующий ядра GPU, не должен иметь выхода во внешнюю сеть или доступа к firmware |
| **HCI / Человеко-машинное взаимодействие** | Проектирование human-on-the-loop интерфейсов для 16+ часовых автономных сессий: как прервать, откатить, аудировать без потери контекста | Интерфейс должен позволять оператору понять состояние агента за <5 минут после пробуждения |
| **Сетевая безопасность**                   | Privilege escalation prevention: формальная модель прав доступа агента к файловой системе, API, другим агентам | Агент не должен иметь возможности grant себе новые права через prompt injection или code execution |
| **Надёжность ПО**                          | Формальная верификация agentic orchestration frameworks (LangChain, CrewAI аналоги) на предмет deadlock, livelock, cascading failures | Ошибка в одном агенте не должна каскадировать на swarm       |

### 4. Верификация результатов автоматизированного ИИ-R&D

| Дисциплина                               | Конкретная научно-инженерная задача                          | Критерий/риск                                                |
| ---------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Автоматическое доказательство теорем** | Интеграция LLM с proof assistants (Lean 4, Coq, Isabelle) для верификации математических результатов, генерируемых ASI | Результат должен иметь machine-checkable proof, а не только «убедительный аргумент» |
| **Формальные методы / CS**               | Построение verified compilers и runtime environments, в которых код, сгенерированный ИИ, выполняется с гарантиями корректности | Использование CompCert-подхода для AI-generated kernels      |
| **Статистика / Эпистемология**           | Разработка calibration metrics для superhuman claims: как измерить confidence в утверждении, которое человек не способен проверить | Необходимы external validation через independent AI systems или physical experiments |
| **Репликабельность науки**               | Автоматизация end-to-end reproduction pipelines: от LaTeX-статьи до Docker-контейнера с воспроизведённым результатом (расширение CoreBench) | Время воспроизведения ML-статьи агентом должно сократиться с недель до часов |

### 5. Безопасность критической инфраструктуры от ИИ-агентов

| Дисциплина                                 | Конкретная научно-инженерная задача                          | Критерий/риск                                                |
| ------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Кибербезопасность / Offensive security** | Создание AI-powered red teams, способных находить уязвимости быстрее, чем autonomous attackers (Project Glasswing-аналоги) | Defensive AI должен опережать offensive AI хотя бы на один порядок по времени патчинга |
| **Формальная верификация ОС**              | Полная формальная верификация ядер ОС (seL4-масштаб) и критических библиотек (OpenSSL, systemd) для устранения класса zero-day | 27-летний баг в OpenBSD — пример неформальной верификации    |
| **Криптография**                           | Разработка AI-resistant криптографических протоколов: какие схемы устойчивы к brute-force и heuristic attacks сверхчеловеческого уровня | Оценка сложности взлома при наличии агента, способного генерировать миллионы эксплойтов в час |
| **SCADA / ICS Security**                   | Защита энергосетей и промышленных контроллеров от autonomous lateral movement: сегментация сети с zero-trust для ИИ-агентов | AlphaEvolve оптимизирует power grids — та же модель может искать уязвимости |

## II. Экономические и институциональные домены

| -------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Экономика труда**              | Построение моделей substitution elasticity между ИИ-агентами и человеческим трудом по секторам (Cobb-Douglas с AI-капиталом) | Нужны эмпирические оценки: при каком уровне SWE-bench автоматизация вытесняет 50% junior-разработчиков |
| **Механизм дизайн / Теория игр** | Проектирование механизмов распределения (UBI, UBA, dividend models), сохраняющих стимулы к инновациям при полной автоматизации труда | Модель должна быть proof-of-concept в sandbox-экономике      |
| **Корпоративное право**          | Разработка legal frameworks для zero-human LLC: кто несёт ответственность, как происходит голосование акционеров, каковы reporting obligations | Первая $1B AI-корпорация требует legal prototype             |

### 7. Переобучение и перераспределение рабочей силы

| Дисциплина                       | Конкретная научно-инженерная задача                          | Критерий/риск                                                |
| -------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Наука об образовании**         | Создание adaptive learning systems, способных в реальном времени перестраивать учебные траектории под half-life of skills (который сокращается с лет до месяцев) | Платформа должна обучать новой профессии за <6 месяцев       |

| **HCI / Инженерия интерфейсов**  | Разработка human-AI complementarity interfaces: не замена человека, а augmentation — где ИИ делает 90%, человек делает критические 10% | Необходимы метрики complementarity, а не только productivity |

### 9. Закон Амдала для экономики

| ----------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |

| **Гражданское строительство**       | Модульное и роботизированное строительство: сокращение цикла «проект → разрешение → возведение» с лет до месяцев | Физические ограничения: земля, материалы, регуляторика       |

### 10. Формирование «машинной экономики»

## IV. Эпистемологические и когнитивные домены

### 14. Потеря способности «понимать» прогресс (Loss of the Plot)

| --------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Когнитивная наука / HCI** | Разработка cognitive scaffolding interfaces: визуализация и «перевод» сверхчеловеческих рассуждений в человекочитаемые ментальные модели | Человек должен понимать «почему» решение ASI, не понимая «как» |
| **Explainable AI (XAI)**    | Построение explanation methods для superhuman systems, где объяснение не упрощает до ложности | LIME/SHAP не работают для emergent reasoning; нужны новые методы |
| **Наука об образовании**    | Эпистемическая грамотность (epistemic literacy): обучение людей оценивать доверие к ИИ-генерируемым знаниям | Курсы по критическому мышлению в эпоху ASI                   |

### 15. Прогнозирование пост-RSI мира

| ---------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |

| **Футурология / Сценарное планирование** | Разработка радикально-uncertain scenario planning frameworks: планирование при отсутствии исторических аналогов | Методы должны быть robust к «unknown unknowns»               |

### 16. Творческие vs инкрементальные способности ИИ

| --------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
