# Reqber: Ваш ИИ-ассистент для научных открытий

## Что такое Reqber?

**Reqber** — это интеллектуальная платформа, которая помогает учёным, исследователям и инноваторам генерировать научные гипотезы и находить неочевидные решения сложных задач.

Представьте, что у вас есть команда из 4 талантливых исследователей, которые:
- Анализируют вашу проблему с разных сторон
- Ищут похожие решения в других областях науки
- Применяют проверенные методы изобретательства
- Проверяют идеи на противоречия

Reqber делает всё это автоматически с помощью искусственного интеллекта.

## Зачем это нужно?

### Проблема, которую решает Reqber

Когда вы сталкиваетесь со сложной научной или технической задачей, обычно происходит следующее:

1. **Вы застреваете в привычных паттернах мышления**
   - Инженер видит инженерное решение
   - Биолог видит биологическое решение
   - А вы упускаете идеи из других областей

2. **Не хватает времени на полный анализ**
   - Чтобы провести полноценный литературный обзор, нужны недели
   - Чтобы изучить 40 принципов изобретательности TRIZ — месяцы
   - А решение нужно уже завтра

3. **Сложно проверить, хорошая ли идея**
   - Генерировать идеи легко
   - А оценить их научную обоснованность — сложно

| Ищете статьи вручную | AI ищет 200+ млн статей за секунды |

## Основной функционал

### 1. One-Shot Discovery ("Решение за один раз")

**Что делает:** Полный цикл исследования одной командой

**Когда использовать:**
- У вас есть конкретная проблема
- Нужно быстро получить набор гипотез
- Хотите понять, что уже известно в этой области

**Пример:**
```
Вы: "Как увеличить срок службы батареи?"

Reqber:
1. Анализирует структуру проблемы
2. Ищет похожие исследования (200+ млн статей)
3. Генерирует 5-10 научных гипотез
4. Оценивает каждую по шкале достоверности
5. Показывает, какие методы использовались

Результат через 30 секунд:
- Гипотеза 1: Новый материал электрода (92% уверенность)
- Гипотеза 2: Биомиметическая структура (88% уверенность)
- Гипотеза 3: Протокол динамической зарядки (79% уверенность)
```

### 2. Multi-Agent Discovery ("Мнение команды экспертов")

**Что делает:** Запускает 4 специализированных AI-агента, которые обсуждают проблему

**Агенты:**
1. **Аналитик** — разбирает проблему на составляющие
2. **Учёный** — генерирует гипотезы
3. **Критик** — находит слабые места
4. **Синтезатор** — объединяет лучшие идеи

**Когда использовать:**
- Проблема сложная и многогранная
- Нужна глубокая проработка
- Важно учесть разные точки зрения

**Пример:**
```
Аналитик: "Выявлено 3 ключевых ограничения..."
Учёный: "Генерирую 12 гипотез с использованием C4+TRIZ..."
Критик: "7 из 12 имеют ограничения: X, Y, Z..."
Синтезатор: "Комбинирую лучшие элементы в 5 рекомендаций..."
```

### 3. C4 Geometry (Когнитивная геометрия)

**Что делает:** Систематизирует мышление через 27 когнитивных состояний

**Объяснение простыми словами:**

Представьте, что мышление — это куб 3×3×3. Каждая клетка — это способ смотреть на проблему:

- **Время:** Прошлое → Настоящее → Будущее
- **Масштаб:** Конкретное → Абстрактное → Мета
- **Агенство:** Я → Другие → Система

**Зачем это нужно:**
- Выход из привычных паттернов мышления
- Поиск неочевидных решений
- Систематический подход к творчеству

**Пример:**
```
Обычное мышление: "Как я (сейчас) могу улучшить батарею?"

C4 помогает задать другие вопросы:
- "Как это делали в прошлом?" (Прошлое)
- "Какие общие законы управляют этим?" (Абстрактное)
- "Как система будет развиваться?" (Будущее)
- "Что делают другие в этой области?" (Другие)
```

### 4. TRIZ (Теория решения изобретательских задач)

**Что делает:** Применяет 40 проверенных принципов изобретательности

**Объяснение простыми словами:**

TRIZ — это набор правил, которые использовали создатели патентов. Оказывается, большинство изобретений используют одни и те же 40 принципов.

**Как работает:**
1. Вы описываете противоречие
   - "Хочу увеличить скорость, но при этом ухудшается точность"

2. Reqber находит принципы, которые решали похожие противоречия
   - Принцип 1: Сегментация
   - Принцип 19: Периодическое действие
   - Принцип 35: Изменение параметров

3. Показывает примеры из других областей
   - "В авиации используют турбулентность для...
   - "В биологии есть аналог: ..."

### 5. Analogy Engine (Поиск аналогий)

**Что делает:** Находит решения из других областей науки

**Пример:**
```
Ваша проблема: Оптимизация сети поставок

Reqber находит аналогии:
- Из биологии: как муравьи находят кратчайший путь
- Из физики: потоки в гидродинамике
- Из нейронаук: как мозг оптимизирует связи
```

### 6. Research Search (Поиск исследований)

**Что делает:** Ищет по 200+ миллионам научных статей

**Источники:**
- Semantic Scholar (200+ млн статей)
- arXiv (2+ млн препринтов)
- Патенты (100+ млн)

**Особенности:**
- Не просто поиск по ключевым словам
- Понимает смысл запроса
- Ранжирует по релевантности
- Показывает контекст

### 7. Validation (Валидация)

**Что делает:** Помогает спланировать и провести эксперименты

**Функционал:**
- Создание экспериментальных протоколов
- Отслеживание наблюдений
- Статистический анализ результатов
- Документирование выводов

## Как использовать Reqber

### Способ 1: Через Web-интерфейс (самый простой)

**Шаг 1:** Откройте браузер и перейдите на `http://localhost:3000`

**Шаг 2:** Выберите режим
- "New Discovery" — для быстрого анализа
- "Multi-Agent" — для глубокой проработки

**Шаг 3:** Опишите проблему
```
Пример хорошего описания:
"Как увеличить срок службы литий-ионных батарей 
при сохранении скорости зарядки?"

Пример плохого описания:
"Батареи плохие, надо улучшить"
```

**Шаг 4:** Анализируйте результаты
- Смотрите на уверенность (confidence score)
- Читайте объяснения методов
- Переходите к источникам

### Способ 2: Через TUI (для любителей терминала)

**Зуск:**
```bash
reqber init                 # Интерактивный мастер настройки (анимация куба + wizard)
reqber tui --cyberpunk      # Полноэкранный TUI в Neon Noir теме
reqber tui --layout turbo   # Режим диссертации: 4 агента + Crisis Radar + GPU
```

**Управление (Muscle Memory Triangle):**
- `↑/↓` или `j/k` — навигация
- `Enter` — выбор / вход
- `Esc` — назад / выход
- `Tab` — переключение панелей
- `C` — Config, `G` — GPU, `R` — Radar, `T` — Travel (time-travel), `?` — Help

**Лейауты (автоматически под терминал):**
| Лейаут | Ширина | Что видно |
|--------|--------|-----------|
| Minimal | <80 | Метрики + ввод (для SSH с телефона) |
| Standard | 80–120 | + Куб-маскот, история чата |
| Deep-Work | 120–180 | + Шаги пайплайна, Crisis Radar, карта контекста |
| Turbo | 180+ | + 4 панели агентов, синтез, GPU-монитор |

**Куб-маскот (8-bit ASCII):**
Состояние куба показывает, что происходит:
- `▫▫▫` — ждёт (пульсирует зелёным)
- `▫◈▫` — думает (мигает голубым)
- `◈▣▫` — обрабатывает (жёлтый прогресс)
- `◈▣◈` — открытие! (золотой + вспышка)
- `✖▫▫` — ошибка (красный + глитч)
- `◈▣◈ ✓` — готово (зелёное затухание)

**Crisis Radar:**
Виджет «Радар человеческих проблем» — агрегирует RSS (HN, arXiv, Reddit, Nature, LessWrong) и ранжирует по срочность × соответствие C4 × новизна. Показывает, какие глобальные вызовы могут быть исследованы прямо сейчас.

**Преимущества TUI:**
- Полноэкранный Rich-интерфейс (не textual — zero deadlock)
- 60fps анимации (MatrixRain, Glitch, Scanline)
- Sparklines в реальном времени (токены, стоимость, задержка)
- Time-travel debug — прыжок к любому событию сессии (`/travel <index>`)
- Адаптивный размер под любой терминал

### Способ 3: Через CLI (для автоматизации)

**Примеры команд:**

```bash
# Быстрое решение одной командой
turbo solve "оптимизация нейросети"

# Мульти-агентный анализ
turbo discover "улучшение батареи"

# Поиск в литературе
turbo research semantic "quantum computing"

# C4 анализ
turbo c4 states

# TRIZ решение противоречия
turbo triz solve --improving "скорость" --worsening "точность"
```

## Как читать результаты

### Confidence Score (Уверенность)

```
90-100%: Высокая уверенность
- Много подтверждающих источников
- Сильная теоретическая база
- Проверено на практике

70-89%: Средняя уверенность
- Есть подтверждающие данные
- Некоторые ограничения
- Требуется дополнительная проверка

50-69%: Низкая уверенность
- Интересная идея
- Мало подтверждающих данных
- Требуется эксперимент

<50%: Спекуляция
- Новая идея
- Нужна разработка
```

### Методы

- **C4+TRIZ**: Использовалась когнитивная геометрия + принципы изобретательности
- **Analogy**: Найдена аналогия из другой области
- **Multi-Agent**: Результат обсуждения AI-агентов
- **Literature**: Основано на литературном обзоре

## Ограничения (важно понимать)

### Что Reqber НЕ делает

1. **Не заменяет человеческое мышление**
   - Это инструмент, а не замена исследователю
   - Вы принимаете финальные решения

2. **Не гарантирует успех**
   - Confidence score — это оценка, не гарантия
   - Эксперименты всё ещё нужны

3. **Не знает контекста вашей организации**
   - Не учитывает бюджетные ограничения
   - Не знает о политических/организационных факторах

4. **Работает с существующими знаниями**
   - Не создаёт принципиально новые физические законы
   - Комбинирует и применяет известные подходы

## FAQ

**Q: Reqber заменит учёных?**
A: Нет. Это инструмент для усиления возможностей исследователя. Как микроскоп не заменяет биолога, так и Reqber не заменяет учёного.

**Q: Насколько точны предсказания?**
A: Confidence score показывает научную обоснованность, но не гарантирует успех. Эксперименты всё ещё необходимы.

**Q: Можно ли использовать для коммерческих проектов?**
A: Да. Но помните, что результаты основаны на публичных исследованиях.

**Q: Что если Reqber не нашёл решение?**
A: Попробуйте переформулировать проблему. Или разбить её на части. Иногда нужно несколько итераций.

**Q: Какой интерфейс выбрать?**
A: - Web — для визуального анализа
  - TUI — для быстрой работы
  - CLI — для автоматизации и скриптов
