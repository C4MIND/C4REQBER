- Режимы: активные (91.9%), постепенные (8.1%)

## ФАЗА 1: Консолидация (резюме)

- Единый реестр собран, минимальные пересечения (только 3 процесса в 2 субдоменах)
- Кросс-связки субдоменов:

  - *Информационный поиск ↔ Информационная грамотность* — навыки работы с информацией
  - *Цифровые библиотеки ↔ Цифровое сохранение* — управление цифровым контентом
  - *Управление знаниями ↔ Библиометрия* — анализ и оценка знаний
  - *Метаданные ↔ Все субдомены* — универсальная инфраструктура описания

## ФАЗА 3: Абстрагирование (по кластерам)

### A) Организующие

- **Мета-паттерн:** Превращение хаоса информации в упорядоченное знание
- **Инварианты:** Схема, иерархия, отношения
- **Вариативность:** Глубина и гибкость структуры
- **Открытия:** Структура доминирует (45.9%) — ключ к управлению информацией

### B) Поисковые

- **Мета-паттерн:** Обеспечение доступа к релевантной информации
- **Инварианты:** Запрос → обработка → результат → оценка
- **Вариативность:** Точность vs полнота
- **Открытия:** Адаптация — ключевой принцип поиска (20 процессов)

### C) Аналитические

- **Мета-паттерн:** Извлечение скрытых паттернов и знаний из данных
- **Инварианты:** Данные → анализ → инсайты → визуализация
- **Вариативность:** Глубина анализа и методы
- **Открытия:** Библиометрия создаёт метазнание о знании

### D) Сохраняющие

- **Мета-паттерн:** Обеспечение долговременной доступности информации
- **Инварианты:** Capture → preserve → maintain → access
- **Вариативность:** Стратегии миграции и эмуляции
- **Открытия:** Устойчивость — самый частый принцип (33 процесса)

### E) Обучающие

- **Мета-паттерн:** Развитие информационных компетенций пользователей
- **Инварианты:** Оценка → обучение → практика → рефлексия
- **Вариативность:** Уровни информационной грамотности
- **Открытия:** Критичность — ключ к информационной грамотности

### F) Трансформирующие

- **Мета-паттерн:** Преобразование информации для новых контекстов использования
- **Инварианты:** Исходная форма → процесс → новая форма
- **Вариативность:** Степень автоматизации
- **Открытия:** 99.6% обратимость позволяет экспериментировать

- **Уникальные (один субдомен):** 277 из 280 процессов (98.9%) — высочайшая специализация

### 6.2. Плейбуки композиций (типовые маршруты)

> **Композиционные паттерны:**
>
> | Сценарий                            | Маршрут (микро-синтаксис)                                    | Цель                      |
> | ----------------------------------- | ------------------------------------------------------------ | ------------------------- |
> | Каталогизация                       | `Acquire → Describe → Classify → Index → Enrich → Publish`   | Ввод в систему            |
> | Информационный поиск                | `Need → Query → Retrieve → Rank → Filter → Evaluate → Refine` | Найти релевантное         |
> | Библиометрический анализ            | `Collect → Normalize → Measure → Cluster → Map → Interpret`  | Анализ науки              |
> | Цифровое сохранение                 | `Identify → Capture → Validate → Migrate → Replicate → Monitor` | Долговременность          |
> | Обучение информационной грамотности | `Assess → Design → Teach → Practice → Evaluate → Iterate`    | Развитие навыков          |
> | Управление знаниями                 | `Capture → Codify → Store → Share → Apply → Update`          | Организационная память    |
> | Создание онтологии                  | `Conceptualize → Structure → Formalize → Validate → Implement → Maintain` | Модель предметной области |
> | Linked Data публикация              | `Model → Convert → Link → Enrich → Validate → Publish → Monitor` | Семантический веб         |

### 6.3. Покрытие (правило Парето)

- **Минимальный набор операторов**, покрывающий ≥80% сценариев: `Classify`, `Index`, `Query`, `Retrieve`, `Rank`, `Capture`, `Enrich`
- **Редкие, но критичные:**
  - `Disambiguate` — разрешение неоднозначности (NLP, семантика)
  - `Deduplicate` — устранение дубликатов (качество данных)
  - `Crosswalk` — маппинг между схемами (интероперабельность)
