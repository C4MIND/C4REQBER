**Направления/полюса** внутри кластеров:

- *Усиление ↔ Ослабление* сигналов (нормализация, фильтрация, регуляризация).
- *Связывание ↔ Разрыв* (интеграция сетов, дедупликация, ансембли против декорреляции фич).
- *Локализация ↔ Генерализация* (локальные окна, attention-маскирование ↔ глобальные пулинги/агрегации).
- *Централизация ↔ Децентрализация* (централизованный тренинг ↔ федеративный/edge).
- *Манифестация ↔ Латентность* (вывод интерпретаций ↔ скрытые стейты/вариационные представления).

## ФАЗА 3: Абстрагирование

### A. Диагностические процессы

- **Мета-паттерн:** *Снятие неопределённости данных/сигнала* (распределения, стационарность, зависимости, ошибки измерения).
- **Инварианты:** воспроизводимые проверки (тест→верификация→контроль), явные предпосылки (i.i.d., марковость, идентифицируемость).
- **Вариативность:** оффлайн EDA ↔ онлайн health-checks; точные тесты ↔ эвристики; локальные окна ↔ глобальная оценка.
- **Открытие:** выбранный диагностический набор **предопределяет** класс допустимых оптимизаторов/архитектур (что меряешь — тем управляешь).

### B. Регулятивные процессы

- **Мета-паттерн:** *Контуры отрицательной/положительной обратной связи* (качество↔стоимость↔латентность).
- **Инварианты:** циклы «мониторинг → коррекция → стабилизация»; контроль ресурсоёмкости (память/время/энергия).
- **Вариативность:** жёсткие/мягкие ограничения (клиппинг/weight decay, квантование/прунинг), batch ↔ streaming.
- **Открытие:** регулятивные приёмы — основной «безопасный слой» необратимых шагов (смягчают потери информации).

### C. Трансформационные процессы

- **Мета-паттерн:** *Переконфигурация представлений/функций* (фичеринг, эмбеддинги, оптимизация, архитектурные мутации).
- **Инварианты:** сохранение валидности таргета и метрик; инварианты симметрий (сдвиг/масштаб/перестановка).
- **Вариативность:** символические/нейронные/гибридные представления; локальный ↔ глобальный контекст; off-policy/on-policy апдейты.
- **Открытие:** сильные представления **снижают** потребность в сложных регулятивных костылях (меньше «ручной» стабилизации).

### D. Интегративные процессы

- **Мета-паттерн:** *Сборка целостности* (ансамбли, трансфер, мультимодальность, федеративность).
- **Инварианты:** совместимость пространств (размерности/масштабы/кодировки), согласование распределений.
- **Вариативность:** bagging/boosting/stacking; ранний/поздний фьюжн; централизованный ↔ федеративный.
- **Открытие:** интеграция работает устойчиво только при **контрактности** интерфейсов (схемы, версии, метаданные).

### E. Коммуникативные процессы

- **Мета-паттерн:** *Переупаковка информации* для людей/сервисов (карты атрибуций, карточки датасетов, отчёты SLI/SLO).
- **Инварианты:** трассируемость, воспроизводимость, контекст предпосылок.
- **Вариативность:** человеко-чит./машино-чит.; статические отчёты ↔ интерактивные панели.
- **Открытие:** коммуникативные артефакты резко снижают «скрытую необратимость» (ошибки повторения экспериментов).

### F. Институциональные процессы

- **Мета-паттерн:** *Фиксация/нормирование изменений* (политики качества данных, аудит экспериментов, безопасность и этика).
- **Инварианты:** версии, подписи, регламенты; внешняя проверка (review).
- **Вариативность:** легковесные гайды ↔ формальные стандарты; on-prem ↔ облачные требования; локальные ↔ отраслевые нормы.
- **Открытие:** без институционализации интеграция и регуляция быстро деградируют под давлением дрифта/техдолга.

## ФАЗА 4: Иерархия процессов

### 4.1 Базовые (универсальны)

- Профилирование/валидация данных; нормализация/очистка; фичеринг/эмбеддинги; обучение/регуляризация; калибровка/валидация; версионирование/трассировка; мониторинг/алерты.

### 4.2 Общие (в большинстве субдоменов)

- Аугментации; ансамбли/стэкинг; дистилляция; отбор признаков; поиск гиперпараметров; отчётность метрик; контроль дрифта/роллбэк.

### 4.3 Специализированные (2–3 субдомена)

- Attention-маскирование (NLP↔CV), сообщение по графу (Graph↔DL), контрфактические объяснения (XAI↔Causal), вариационные стейты (DL↔TS), ИВС/IV-подходы (TS↔Causal).

### 4.4 Уникальные (узкие)

- Идентифицируемость причин (Causal), стационаризация/сезонное разложение (TS), NMS/IoU-метрики (CV), BPE/Tokenizer-процедуры (NLP), SHAP/IG/TCAV (XAI), сообщение по топологиям (Graph).

## ФАЗА 5: Фундаментальные мета-принципы (5–10)

1. **Необратимость по умолчанию.** Большинство шагов теряют информацию (агрегирование, регуляризация, стох. оптимизация); обратимость — исключение. Следствие: критичны диагностика и регуляция до и после трансформаций.
2. **Контрактность интерфейсов.** Схемы/версии/метаданные — условие переносимости и интеграции; без контрактов любая интеграция хрупка.
3. **Идентифицируемость прежде оптимальности.** Без выполнимых предпосылок (стационарность, инструментальность, acyclicity) оптимизация даёт иллюзию прогресса.
4. **Баланс «представление ↔ регуляризация».** Чем богаче представление, тем меньше требуется жёсткой регуляции; но без регуляции возрастает риск переобучения/стоимости.
5. **Мультишкальность и топология.** Результаты чувствительны к масштабу (окна/рецептивные поля) и структуре (графы/архитектуры); topology-aware операции повышают устойчивость.
6. **Время как скрытый модератор.** Нестационарность/дрифт/выборка по времени ломают генерализацию; необходимы онлайновые диагностика и адаптация.
7. **Непротиворечивость метрик.** Метрики должны согласовываться с задачей (калибровка/utility-оптимальность/ограничения); иначе операционные решения деградируют.
8. **Причинность > корреляция для действий.** Для интервенций/политик нужны причинные эффекты; корреляции достаточны лишь для статичного предсказания.
9. **Ограничения как ресурс.** Бюджеты вычислений/памяти/энергии/латентности — конструктивные ограничения, задающие пространство допустимых архитектур/алгоритмов.
10. **Этика и безопасность по умолчанию.** Приватность, справедливость, безопасность и объяснимость — не «после этого», а параметры задачи (влияют на выбор трансформеров).

## Операционная «алгебра» домена (канонические ходы)

**Sample / Split / Normalize / Encode / Aggregate / Filter / Difference / Convolve / Optimize / Regularize / Quantize / Prune / Attend / Pool / Propagate / Mask / Augment / Distill / Ensemble / Simulate / Intervene / Estimate / Debias / Calibrate / Attribute / Monitor / Rollback.**
 Типовые композиции:

- **Diagnose → Clean/Normalize → Feature/Embed → Train(Optimize+Regularize) → Validate/Calibrate → Deploy → Monitor → Adapt/Rollback.**
- **Localize(Time/Graph) → Represent(Attention/MP) → Integrate(Ensemble/Fusion) → Explain(XAI) → Govern(Policy/Compliance).**
- **Assume(Test) → Identify(Causal) → Estimate(ATE/CATE) → Stress-test (Counterfactuals) → Deploy (Safeguards).**

------

## Инварианты домена

- Воспроизводимость (версии данных/кода/артефактов), трассируемость (эксперименты/метрики), контракты интерфейсов, явные предпосылки (DGP-гипотезы), безопасные границы (SLO/латентность/ресурсы).

## Параметры вариативности

- Масштаб (локальные окна ↔ глобальные контексты), жёсткость ограничений (float32↔int4, dense↔sparse), онлайновость (batch↔stream), центр/edge (централизация↔федерация), степень гибридности (символика↔нейронка↔каузалити).

## Практическая карта соответствия (shortlist)

1. **Контуры качества:** *Диагностика → Регуляция → Трансформация → Интеграция → Институционализация → Мониторинг*.
2. **Анти-дрифт шлюз:** локализация по времени/сегментам, калибровка, контрактные тесты, автопауза и роллбэк.
3. **Действия под причинностью:** если есть интервенции/политики → встраивайте идентифицируемость и контрфактические проверки до обучения.

## TL;DR

Data Science — домен **преимущественно необратимых** трансформаций данных и представлений; устойчивость достигается через **контракты**, **диагностику/регуляцию**, **сильные представления** и **институционализацию** практик (версии/аудит/политики). Ключевые оси: время/топология/ресурсы/причинность. Операционная алгебра (от Sample до Rollback) обеспечивает маршруты от «сырых данных» к стабильной эксплуатации и осмысленным интервенциям.
