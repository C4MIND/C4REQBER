# c4reqber Formal Proof Corpus

This directory contains formal proofs generated by **c4reqber's AI Meta-Solver**.

## Relationship to Foundational Work

Foundational theorems of the C4 cognitive architecture are proven in the `adaptive-topology` repository. This corpus is distinct: it holds **new theorems discovered or generated during live c4reqber sessions**, not copies of existing foundational results.

## Purpose

New theorems here are discoveries made during c4reqber sessions. They capture emergent properties, lemmas, and corollaries that arise from the meta-solver's exploration of the C4 cognitive architecture and related formal systems.

## Directory Structure

```
formal-proofs/
├── lean4/         -- Lean 4 proof scripts
├── coq/           -- Coq proof scripts
├── dafny/         -- Dafny verification-aware programs
├── agda/          -- Agda dependently-typed proofs
├── hoare/         -- Hoare-logic / imperative verification specs
└── discoveries/   -- New theorem discovery bundles
```

## Naming Convention

All proof files follow the pattern:

```
NNNN-theorem-name.ext
```

- `NNNN` — zero-padded sequence number (0001, 0002, ...)
- `theorem-name` — kebab-case descriptive name
- `ext` — language-specific extension (`.lean`, `.v`, `.dfy`, `.agda`, `.py`)

The `0000-template.ext` files in each subdirectory are reserved as reference templates and should not be renamed or removed.

## How to Contribute / Add Proofs

1. Choose the appropriate language directory for your proof.
2. Copy the `0000-template.ext` file to a new file with the next sequence number and a descriptive name.
3. Fill in the header comment with date, author, and theorem statement.
4. Replace the placeholder with the actual proof or `sorry`/`Admitted`/equivalent for partial work.
5. For new discoveries, also create a dated entry under `discoveries/YYYY-MM-DD-theorem-name/`.
6. Verify the proof using the language-specific instructions in the subdirectory README.
7. Ensure all placeholders are resolved before marking a proof as complete.
